Mathematics 1 — Formula Bank
Every formula you need, with the logic behind it.
Quadratic Basics
Standard & Vertex Form
Standard: ax² + bx + c
Vertex: a(x - h)² + k
─────────────────────
h = -b/(2a) [axis of symmetry]
k = c - b²/(4a) [vertex y-value]
Why: Completing the square on ax²+bx+c gives a(x+b/2a)²+(c-b²/4a). The vertex is where the parabola turns — slope = 0 there.
Quadratic
Discriminant & Roots
Δ = b² - 4ac
─────────────────────
Δ > 0 → 2 real roots
Δ = 0 → 1 real root (touches)
Δ < 0 → no real roots
─────────────────────
x = (-b ± √Δ) / (2a)
Why: The quadratic formula comes from completing the square. Δ tells you what's under the root — negative means imaginary.
Quadratic
Vieta's Formulas
If roots are α, β:
α + β = -b/a
α × β = c/a
─────────────────────
Sum of roots = -b/a
Product of roots = c/a
Why: a(x-α)(x-β) = ax²-a(α+β)x+aαβ. Matching with ax²+bx+c gives these. No need to find roots individually!
Lines
Line Equations & Slope
Slope: m = (y₂-y₁)/(x₂-x₁)
Slope-intercept: y = mx + c
Point-slope: y-y₁ = m(x-x₁)
General: ax + by + c = 0
─────────────────────
Slope from general: m = -a/b
Why: Slope = rise/run. The general form ax+by+c=0 rearranged is y=(-a/b)x+(-c/b), so slope = -a/b directly.
Lines
Parallel & Perpendicular
Parallel: m₁ = m₂
Perpendicular: m₁ × m₂ = -1
─────────────────────
Line x/a - y/b = 1:
Rewrite → slope = b/a
(multiply both sides by ab)
Why: Perpendicular lines cross at 90°. If one has slope m, the other rotated 90° has slope -1/m. Product = m×(-1/m) = -1.
Coordinate Geometry
Distance & Section Formula
Distance: d = √[(x₂-x₁)²+(y₂-y₁)²]
─────────────────────
Internal division (m:n):
P = ((mx₂+nx₁)/(m+n), (my₂+ny₁)/(m+n))
─────────────────────
Midpoint: ((x₁+x₂)/2, (y₁+y₂)/2)
Why: Section formula splits a segment internally at ratio m:n. Think of it as a weighted average — m pulls toward point 2, n pulls toward point 1.
Polynomials
End Behavior Rules
Degree n, leading coeff a:
─────────────────────
a > 0, n even → ↑ both ends
a < 0, n even → ↓ both ends
a > 0, n odd → ↓left, ↑right
a < 0, n odd → ↑left, ↓right
─────────────────────
Max turning points = n - 1
Why: Only the highest-degree term dominates as x→±∞. Odd powers flip sign, even powers don't. Turning points = where derivative = 0, max is degree-1.
Relations
Equivalence Relation Checklist
Reflexive: (a,a) ∈ R ∀a
Symmetric: (a,b) ∈ R → (b,a) ∈ R
Transitive: (a,b),(b,c) ∈ R → (a,c) ∈ R
─────────────────────
Equivalence = ALL THREE ✓
Why: Equivalence relations partition a set into classes where everything is "equivalent." Like "same level" — it's reflexive, symmetric, transitive.
Sets
Set Operations
|A∪B| = |A| + |B| - |A∩B|
A\B = elements in A but not B
A△B = (A\B) ∪ (B\A)
─────────────────────
Only A = |A| - |A∩B|
Only B = |B| - |A∩B|
Only one = Only A + Only B
Why: Inclusion-exclusion removes double-counting. A\B removes the overlap from A. "Only one subject" means union minus the overlap counted once.
Parabola
Vertex Form → Vertex Reading
y - k = a(x - h)²
─────────────────────
Vertex = (h, k)
─────────────────────
TRICK: h is with x, k is with y
y-6 = 3(x-10)² → vertex (10,6)
NOT (6,10)!
Why: The vertex is where the squared term = 0, i.e. x=h, y=k. Students often flip (h,k) — remember: h goes with x, k goes with y.
Regression
Sum of Squared Errors (SSE)
SSE = Σ(yᵢ - ŷᵢ)²
─────────────────────
ŷ = predicted (from regression line)
y = actual observed value
─────────────────────
Error = actual - predicted
Why: SSE measures total "miss" of the model. Square the errors so negatives don't cancel positives. Given total SSE, subtract known errors² to find missing one.
Polynomials
Roots & X-intercepts
f(x) = a(x-r₁)ⁿ¹(x-r₂)ⁿ² ...
─────────────────────
Distinct x-intercepts = distinct rᵢ
Repeated root = touches, not crosses
─────────────────────
(x-5)²(x-7) → x=5 (touch), x=7 (cross)
Why: A repeated root means the parabola just "bounces" off the x-axis rather than crossing. Distinct intercepts = distinct roots regardless of multiplicity.
Concept Explainers — Question Types
Strip the costume. See the naked idea.
1
Finding f(x) through Intersection Point
The scary version: "f(x) has vertex at intersection of y₁ and y₂, y-intercept is 2..."

The simple version: Find where y₁=y₂ → that gives you (h,k). Write f in vertex form, plug in y-intercept to find 'a'.
STEP 1 — Find Intersection (Vertex)
Set y₁ = y₂. Solve for x. Plug x back into either equation to get y. → Vertex (h, k)
STEP 2 — Build f(x) and find 'a'
Write f(x) = a(x-h)² + k. Plug in the y-intercept point (0, c). Solve for a.
The vertex of f is GIVEN to you as the intersection point — you don't need to guess it. Just find where y₁=y₂.
2
Equivalence Relations — The 3-Property Test
The scary version: "Let R = {(A,B) | A and B are in same level}. Is R an equivalence relation?"

PropertyQuestion to AskExample Check
ReflexiveIs every element related to itself?Is (Maths, Maths) ∈ R? → YES (same level as itself)
SymmetricIf (A,B)∈R, is (B,A)∈R?If Maths~Stats, then Stats~Maths? → YES
TransitiveIf (A,B) and (B,C) in R, is (A,C)?All same level → YES
For S (prerequisite relation): Maths is NOT its own prerequisite → NOT reflexive → NOT an equivalence relation. Always check reflexive first — it's the easiest to fail.
3
Polynomial End Behavior
The scary version: "p(x) = -x⁵ + 5x⁴ - 7x - 2, which statements are true about end behavior?"

The simple version: IGNORE everything except the leading term.
THE ONLY RULE YOU NEED
Leading term = -x⁵. As x→+∞, x⁵→+∞, so -x⁵→-∞. As x→-∞, x⁵→-∞, so -x⁵→+∞.
Memory trick: Negative + Odd = "Starts high, ends low" (goes from +∞ to -∞ left to right). Like a slide going down.
4
Section Formula — Internal Division
The scary version: "P divides MN internally in ratio 2:1, find slope from origin to P."

The simple version: The formula is a weighted average. Ratio m:n means m parts toward point 2, n parts toward point 1.
FORMULA — P divides AB in ratio m:n
P = ( (m·x₂ + n·x₁)/(m+n), (m·y₂ + n·y₁)/(m+n) )
Mnemonic: "m is bigger pull toward the SECOND point." If ratio is 2:1, you're 2/3 of the way from first to second point.
5
Two-Line Intersection Problems
Pattern: Line l₁ with given intercepts → find equation. Find its intersection with another line. New line l₂ passes through intersection with given condition (parallel to axis, etc.).
STEP 1 — Find l₁'s equation
Two intercepts (a,0) and (0,b) → slope = -b/a, equation: x/a + y/b = 1
STEP 2 — Find intersection with another line
Solve as simultaneous equations. The intersection point is on both lines.
STEP 3 — Apply condition for l₂
"Parallel to x-axis" → slope=0 → l₂: y = constant (= y-coordinate of intersection)
Parallel to x-axis = horizontal line = y=k. Parallel to y-axis = vertical line = x=k. The y-intercept of y=k is just k.
6
SSE Regression — Finding Missing Value
Pattern: Given regression line ŷ = mx+c, actual data with one missing value, total SSE = known value. Find the missing actual observation.
STEP 1 — Calculate (actual-predicted)² for all known rows
For each data point: compute ŷᵢ = m·xᵢ + c, then compute (yᵢ-ŷᵢ)² and sum them.
STEP 2 — Use total SSE to find the missing error
(y₅-ŷ₅)² = Total SSE - Sum of known errors². Take square root, apply sign condition.
The condition "actual EXCEEDED predicted" tells you the sign is positive. So y₅ = ŷ₅ + √(remaining SSE).

Statistics 1 — Formula Bank
Central Tendency
Mean Types
Arithmetic Mean: x̄ = Σxᵢ/n
Weighted Mean: x̄w = Σ(wᵢxᵢ)/Σwᵢ
─────────────────────
Median: middle value (sorted)
Mode: most frequent value
Why weighted mean: Not all values contribute equally. A student scoring 90 in a 4-credit course matters more than 60 in a 1-credit course.
Spread
Variance & Standard Deviation
Population Variance:
σ² = Σ(xᵢ - μ)²/n
─────────────────────
Sample Variance:
s² = Σ(xᵢ - x̄)²/(n-1)
─────────────────────
Shortcut: σ² = (Σxᵢ²/n) - μ²
Why n-1 for sample: Bessel's correction. Sample mean is computed from the data, removing one "degree of freedom." Dividing by n would underestimate true variance.
Position
Percentiles & Quartiles
Pₖ position = k(n+1)/100
Q1 = P₂₅, Q2 = P₅₀, Q3 = P₇₅
IQR = Q3 - Q1
─────────────────────
Outlier if:
x < Q1 - 1.5×IQR OR
x > Q3 + 1.5×IQR
Why 1.5×IQR: Tukey's fence. For normally distributed data, ~0.7% of values fall outside. It's the standard "outlier detector" in box plots.
Regression
Simple Linear Regression
ŷ = b₀ + b₁x
b₁ = [nΣxy - ΣxΣy] / [nΣx² - (Σx)²]
b₀ = ȳ - b₁x̄
─────────────────────
SSE = Σ(yᵢ - ŷᵢ)²
SST = Σ(yᵢ - ȳ)²
R² = 1 - SSE/SST
Why this formula for b₁: Minimizing SSE (least squares). Taking derivative of SSE w.r.t b₁ and setting to 0 gives this exact formula.
Probability
Basic Probability Rules
P(A∪B) = P(A)+P(B)-P(A∩B)
P(A|B) = P(A∩B)/P(B)
P(A∩B) = P(A)×P(B) [if independent]
P(Aᶜ) = 1 - P(A)
─────────────────────
Bayes: P(A|B) = P(B|A)P(A)/P(B)
Why inclusion-exclusion: P(A)+P(B) double-counts the intersection, so subtract it once. Same logic as set cardinality formula.
Correlation
Pearson Correlation
r = Σ(xᵢ-x̄)(yᵢ-ȳ) / √[Σ(xᵢ-x̄)²·Σ(yᵢ-ȳ)²]
─────────────────────
-1 ≤ r ≤ 1
r=1: perfect positive
r=-1: perfect negative
r=0: no linear correlation
Why between -1 and 1: By Cauchy-Schwarz inequality. r is a normalized covariance — dividing by both standard deviations scales it to [-1,1].
Statistics 1 — Concept Explainers
1
Weighted Mean — When values have different importance
Like CGPA — a 4-credit course weighs more than a 1-credit course.
FORMULA IN ACTION
Courses: 90(4cr), 80(3cr), 70(2cr)
CGPA = (90×4 + 80×3 + 70×2)/(4+3+2) = (360+240+140)/9 = 740/9 ≈ 82.2
Always divide by SUM of weights, NOT by count of items. This is the most common mistake in weighted mean problems.
2
Outlier Detection with IQR Method
Step-by-step process:
STEP 1 — Sort data and find Q1, Q3
For n data points: Q1 is at position (n+1)/4, Q3 is at 3(n+1)/4. Interpolate if needed.
STEP 2 — Compute IQR and fences
IQR = Q3 - Q1
Lower fence = Q1 - 1.5×IQR
Upper fence = Q3 + 1.5×IQR
STEP 3 — Flag outliers
Any value outside [lower fence, upper fence] is an outlier.
The fences are NOT the outliers themselves — values BEYOND the fences are outliers.
3
Variance Shortcut Formula
The ugly way: σ² = Σ(xᵢ - μ)²/n (subtract mean from each, square, average)
The smart way: σ² = (Σxᵢ²/n) - μ² (mean of squares minus square of mean)
EXAMPLE
Data: 2, 4, 6. Mean = 4.
Smart: (4+16+36)/3 - 4² = 56/3 - 16 = 18.67-16 = 2.67
Check: ((2-4)²+(4-4)²+(6-4)²)/3 = (4+0+4)/3 = 2.67 ✓
Shortcut formula avoids computing each (xᵢ-μ)² separately. Much faster in exams! Just compute Σxᵢ² and μ².

Computational Thinking — Core Concepts
Logic
Boolean Truth Tables
AND (∧): True only if BOTH true
OR (∨): True if AT LEAST ONE true
NOT (¬): Flips the value
XOR: True if EXACTLY ONE true
─────────────────────
De Morgan's:
¬(A∧B) = ¬A ∨ ¬B
¬(A∨B) = ¬A ∧ ¬B
Memory trick for De Morgan's: "Break the bar, change the operator." NOT(A AND B) = (NOT A) OR (NOT B).
Algorithms
Loop Patterns
FOR i FROM 1 TO n:
  → runs n times
─────────────────────
WHILE condition:
  → runs until condition false
─────────────────────
Accumulator: sum = sum + x
Counter: count = count + 1
Flag: found = True/False
Trace tables: Always track variable state at each iteration. Most CT exam errors come from off-by-one — trace carefully at loop start and end.
Pseudocode
Pseudocode Conventions
INPUT x ← reads value
OUTPUT/PRINT x ← displays value
← or = ← assignment
IF...THEN...ELSE...END IF
FOR...TO...NEXT or END FOR
WHILE...DO...END WHILE
FUNCTION...RETURN
Key insight: In pseudocode, = is usually assignment, not equality. Equality comparison is often == or "equals". Read carefully!
Data Structures
Arrays & Lists
Array index: usually 0 or 1-based
Length of arr: n elements
─────────────────────
Access: arr[i]
Iterate: FOR i FROM 0 TO n-1
─────────────────────
Stack: LIFO (last in, first out)
Queue: FIFO (first in, first out)
Common trap: If 0-indexed, last element is at index n-1, not n. If the pseudocode says arr[1..n], it's 1-indexed.
Complexity
Time Complexity Basics
O(1) → constant (no loop)
O(n) → one loop over n
O(n²) → nested loops
O(log n) → halving each step
O(n log n) → merge/quick sort
─────────────────────
O(1) < O(log n) < O(n) < O(n²)
Quick check: Count nested loop levels. 1 level = O(n), 2 levels = O(n²). Binary search halves each time → O(log n).
Functions
Recursion Pattern
FUNCTION fact(n):
  IF n = 0: RETURN 1
  ELSE: RETURN n × fact(n-1)
─────────────────────
Base case: stops recursion
Recursive case: calls itself
─────────────────────
fact(4) = 4×3×2×1×1 = 24
Always trace: Start from the call and expand. fact(3) = 3×fact(2) = 3×2×fact(1) = 3×2×1×fact(0) = 3×2×1×1 = 6.

PRINT sum

Always build the trace table. Never try to "calculate in your head" — that's where mistakes happen.
2
Boolean Flag Pattern
Flags are used to track whether a condition was EVER true during a loop.
SET found ← False
FOR i FROM 1 TO n DO
  IF arr[i] = target THEN
    found ← True
  END IF
END FOR
IF found THEN PRINT "Found"
Flag starts False. If the condition hits at ANY point, it becomes True and stays True. Used for: "does any element satisfy X?" type questions.

English 1 — Rules & Reference
Grammar
Subject-Verb Agreement
Singular subject → singular verb
Plural subject → plural verb
─────────────────────
Tricky cases:
Each/Every/Either/Neither → singular
None → can be singular or plural
Collective nouns → usually singular
"The number of" → singular
"A number of" → plural
Test tip: Ignore phrases between subject and verb. "The group of students [IS/ARE] here" — subject is "group" (singular) → IS.
Grammar
Tenses Quick Reference
Simple Past: V₂ (went, took)
Past Continuous: was/were + Ving
Present Perfect: has/have + V₃
Past Perfect: had + V₃
─────────────────────
Signal words:
"since/for/already" → Present Perfect
"when/while" → Past Continuous
"by the time/before" → Past Perfect
Key distinction: Present Perfect connects past to now ("I have eaten" — still full). Simple Past is done and over ("I ate at 2pm").
Vocabulary
High-Frequency Word Pairs
Affect (verb) vs Effect (noun)
Principle (rule) vs Principal (head)
Complement (complete) vs Compliment (praise)
Eminent (famous) vs Imminent (soon)
Elicit (draw out) vs Illicit (illegal)
Stationary (still) vs Stationery (paper)
Affect/Effect trick: "RAVEN" — Remember Affect Verb Effect Noun. Principal has an "pal" (person). StationEry has an E like pEn.
Idioms
Common Idioms for IITM English
"Hit the nail on the head" = correct
"Bite the bullet" = endure hardship
"Beat around the bush" = avoid topic
"Break the ice" = ease tension
"Burn bridges" = end relationships
"Cost an arm and a leg" = very expensive
"In the same boat" = same situation
"Pull someone's leg" = joke/tease
Test strategy: Idiom meanings are never literal. "Kick the bucket" ≠ kicking anything. Always pick the figurative/metaphorical meaning.
Phrasal Verbs
Essential Phrasal Verbs
Call off = cancel
Call for = require/demand
Give up = stop trying
Give in = surrender/yield
Look up = search/research
Look into = investigate
Put off = postpone
Put up with = tolerate
Run out of = exhaust supply
Turn down = reject/refuse
Pattern: Same verb + different preposition = completely different meaning. "Give up" vs "Give in" vs "Give out" are all different. Learn them as fixed phrases.
Reading
RC Passage Strategy
1. Read question FIRST
2. Skim passage for structure
3. Read relevant section carefully
4. Answer from passage, not opinion
─────────────────────
Wrong answer traps:
- True in general, not in passage
- Extreme words (always, never)
- Half-right answers
- Opposite of what passage says
Golden rule: The answer must be directly supported by text in the passage. If you're inferring more than one logical step, it's probably wrong.
English 1 — Concept Explainers
1
Spotting Grammar Errors — The Scan Method
When asked to find an error in a sentence, scan in this order:
SCAN ORDER
1. Subject-verb agreement
2. Tense consistency
3. Article usage (a/an/the)
4. Preposition usage
5. Pronoun agreement
80% of IITM English grammar errors fall in steps 1-3. Always check subject-verb first — it's the most tested.
2
Vocabulary in Context — Elimination Strategy
When you don't know a word, use elimination:
4-STEP ELIMINATION
1. Find tone of sentence (positive/negative/neutral)
2. Eliminate options with wrong tone
3. Check grammatical fit (noun/verb/adj)
4. Pick the most contextually precise remaining option
Even with zero vocabulary knowledge, you can often eliminate 2 of 4 options using tone and grammar. That gives you 50% odds minimum.
